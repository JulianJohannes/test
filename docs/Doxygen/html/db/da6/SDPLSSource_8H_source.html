27  Foam::SDPLSSource

29 Description
30  The base class SDPLSSource is the root for the model hierarchy of the SDPLS
31  method by Fricke et. al. \cite fricke_locally_2022.
32  It is the only class with which the user of the SDPLS method should interact.
33  The SDPLS method extends the standard LS advection equation with the SDPLS source term.
34  This is done by extending the r.h.s. of the LS equation in the solver application by calling
35 
36  fvmSDPLSSource(const volScalarField& psi, const volVectorField& U)
37 
38  on the RTS selected model and passing a reference to the LS field psi and the
39  velocity field U. It returns a tmp<fvScalarMatrix> object for integration into
40  the LS equation. This class hierarchy is responsible for providing
41  implementations for the formula of non-linear SDPLS source terms. It is assumed
42  that all SDPLS source term variants have the following structure and evaluate
43  the source in cell \f$\Omega_c\f$ as
44  \f{equation}{ \label{eq:SDPLS source}
45  S_{SDPLS}(\psi^{n+1}) = f_{nl}(\psi^{n+1}) \psi^{n+1}_c
46  \f}
47  with the solution field \f$\psi^{n+1}\f$ at time \f$t^{n+1}\f$, some non-linear
48  scalar function \f$f_{nl}(\psi^{n+1})\f$ and \f$\psi^{n+1}_c\f$ the solution
49  value in cell \f$\Omega_c\f$. All inherited classes are responsible for
50  implementing this non-linear function by overloading the member function
51  nonLinearPart(). This base class implements the case of an inactive source term.
52  The discretization is done separately by the class hierarchy SourceScheme, which
53  is related via the strategy pattern. Furthermore, this class has a strategy for
54  a Mollifier, which restricts the active source term to a narrow band around the
55  interface.
