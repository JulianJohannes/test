\doxysubsection{Detailed Description}
The base class \mbox{\hyperlink{classFoam_1_1SDPLSSource}{SDPLSSource}} is the root for the model hierarchy of the SDPLS method by Fricke et. al. \cite{fricke_locally_2022}. It is the only class with which the user of the SDPLS method should interact. The SDPLS method extends the standard LS advection equation with the SDPLS source term. This is done by extending the r.\+h.\+s. of the LS equation in the solver application by calling. 

\label{_Description}%


\mbox{\hyperlink{classFoam_1_1SDPLSSource_a6b5ad9ef2c399ff536fe684d53fdc168}{fvm\+SDPLSSource(const vol\+Scalar\+Field\& psi, const vol\+Vector\+Field\& U)}}

on the RTS selected model and passing a reference to the LS field psi and the velocity field U. It returns a tmp$<$fv\+Scalar\+Matrix$>$ object for integration into the LS equation. This class hierarchy is responsible for providing implementations for the formula of non-\/linear SDPLS source terms. It is assumed that all SDPLS source term variants have the following structure and evaluate the source in cell $\Omega_c$ as \begin{equation} \label{eq:SDPLS source} S_{SDPLS}(\psi^{n+1}) = f_{nl}(\psi^{n+1}) \psi^{n+1}_c \end{equation} with the solution field $\psi^{n+1}$ at time $t^{n+1}$, some non-\/linear scalar function $f_{nl}(\psi^{n+1})$ and $\psi^{n+1}_c$ the solution value in cell $\Omega_c$. All inherited classes are responsible for implementing this non-\/linear function by overloading the member function \mbox{\hyperlink{classFoam_1_1SDPLSSource_acbfa906d29f0390bb1b529561314da48}{non\+Linear\+Part()}}. This base class implements the case of an inactive source term. The discretization is done separately by the class hierarchy \mbox{\hyperlink{classFoam_1_1SourceScheme}{Source\+Scheme}}, which is related via the strategy pattern. Furthermore, this class has a strategy for a \mbox{\hyperlink{classFoam_1_1Mollifier}{Mollifier}}, which restricts the active source term to a narrow band around the interface.
